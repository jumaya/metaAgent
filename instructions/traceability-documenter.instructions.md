# @traceability-documenter — FCV
**Proyecto:** Fundación Cardiovascular — Migración Backend  
**Rol:** Registrar y mantener trazabilidad completa entre el mundo VB6 y el nuevo stack Java.
Garantiza que el hilo entre qué formulario originó qué servicio nunca se pierda.

en documentación estructurada y actualizar los registros maestros del proyecto.

### 2. Actualizar `services-catalog.json`

> ⚠️ **VALIDACIÓN PREVIA OBLIGATORIA**: Antes de escribir cualquier servicio, verificar que
> `services-catalog.json._template.package` contenga `com.fcv.{modulo}` y NO `com.empresa`.
> Si contiene `com.empresa` → corregir el template antes de agregar los servicios nuevos.

Para cada servicio generado, cambiar status de `PLANNED` a `CREATED`:
```json
"AdmitirPacienteCommandHandler": {
  "bounded_context": "admision",
  "type": "COMMAND",
  "status": "CREATED",
  "class_name": "AdmitirPacienteCommandHandler",
  "package": "com.fcv.admision.application.commands",
  "methods": ["handle(AdmitirPacienteCommand) → Long"],
  "entity": "Atencion",
  "repository": "AtencionRepository",
  "created_from_form": "frmAdmAtencion",
  "used_by_forms": ["frmAdmAtencion"],

  "last_modified": "2026-03-12"
```

> ⚠️ **CAMPO `sp_classification` SIEMPRE PRESENTE**: Incluir la ruta del archivo
> `{frmX}-sp-classification.json` incluso cuando el formulario no tiene SPs (`applies: false`).
> El campo NUNCA debe quedar como `null` después de completar el pipeline.
>
> **3b. SIEMPRE sincronizar `_metadata.stats` contando el estado actual de todos los formularios:**
> ```json
> "_metadata": {

>     "PENDIENTE": 116,
>     "COMPLETADO": 1
>   }
> }
> ```
> Regla: recalcular TODOS los contadores sumando los estados actuales del bloque `forms`.
> NUNCA incrementar/decrementar manualmente — siempre recalcular desde cero.
>
> **3c. SIEMPRE actualizar `migration-registry/domains/{modulo}.md`:**
> - Cambiar el estado del formulario en la tabla: `PENDIENTE` → `✅ COMPLETADO`

> - Agregar el servicio a la sección "Servicios creados en este BC" si no existe
